## ICMP

ICMP (Internet Control Message Protocol) is Layer 3 — it rides inside an IP packet but isn't TCP or UDP. It carries error messages and diagnostic signals between network devices. Crucially, **ICMP has no port numbers** — it's identified by Type and Code fields instead.

### Type 3 — Destination Unreachable

**When triggered:** A router or the destination host cannot deliver the packet and must tell the sender why.

The Type 3 message is generated by a **router or the destination host** (not intermediate devices) and sent back to the **original source**.

Common Code values within Type 3:
| Code | Meaning |
|------|---------|
| 0 | Network unreachable — no route to the destination network |
| 1 | Host unreachable — network reachable but specific host isn't responding |
| 2 | Protocol unreachable — host doesn't support the requested protocol |
| 3 | Port unreachable — host reachable but no service listening on that port (common in UDP scans) |
| 13 | Communication administratively prohibited — a firewall dropped the packet |

**Example:** You send a UDP packet to port 9999 on a host with nothing listening there. The host's OS generates a Type 3 Code 3 (Port Unreachable) ICMP back to you. This is exactly how `nmap` UDP scanning works — silence = possibly open, Type 3 = closed/filtered.

---

### Type 11 — TTL Exceeded

**When triggered:** A router receives a packet whose IP TTL field has reached 0, so it drops the packet and sends a Type 11 back to the sender.

**The TTL field in the IP header** is decremented by 1 at every router hop. It starts at a value like 64 (Linux) or 128 (Windows). When it hits 0 at a router, that router:
1. Drops the packet (doesn't forward it)
2. Generates an ICMP Type 11 Code 0 (Time to Live Exceeded in Transit) message
3. Sends it back to the **original source IP**

The Type 11 message includes the first 8 bytes of the original packet's payload, which is how the source knows which probe triggered the error.

**Why this exists:** Prevents routing loops from circulating packets forever. A TTL of 0 = "this packet has been bouncing around too long, discard it."

## Traceroute mechanics

Traceroute is an elegant hack: it **deliberately exploits TTL Exceeded errors** to map every router hop between you and a destination.

### How TTL expiry is exploited

Traceroute sends probes (UDP packets on Linux/Mac, ICMP Echo Requests on Windows `tracert`) with **artificially low TTL values**, starting at 1 and incrementing.

- TTL=1 → the **first router** decrements it to 0, drops the packet, sends Type 11 back. You learn the first router's IP from the source of that Type 11.
- TTL=2 → the **second router** is the one that hits 0. You get its IP.
- TTL=3 → third router. And so on.
- Eventually TTL is high enough to reach the **destination**. It responds with ICMP Type 3 (Port Unreachable, because traceroute used a random high UDP port) — or ICMP Echo Reply if using ICMP mode. This tells you the path is complete.

### What happens at each hop

```
Your PC (TTL=1) ──→ Router 1: TTL hits 0
                        └──→ Type 11 back to you  [you learn Router 1's IP]

Your PC (TTL=2) ──→ Router 1: TTL=1, forwards
                        └──→ Router 2: TTL hits 0
                               └──→ Type 11 back to you  [you learn Router 2's IP]

Your PC (TTL=3) ──→ Router 1 → Router 2 → Router 3: TTL hits 0
                                               └──→ Type 11 back   [Router 3's IP]

...

Your PC (TTL=N) ──→ ... → Destination: packet arrives
                              └──→ Type 3 Code 3 (Port Unreachable)  [DONE]
```

Traceroute sends **3 probes per TTL** and measures round-trip time for each. If a router doesn't send Type 11 back (firewall blocks ICMP or the router is configured to not respond), you see `* * *` for that hop.

### Which ICMP type comes back and from whom

| Hop state | ICMP type returned | Sent by |
|-----------|-------------------|---------|
| Intermediate router — TTL hits 0 | Type 11 Code 0 (TTL Exceeded) | That router |
| Destination reached (UDP mode) | Type 3 Code 3 (Port Unreachable) | The destination host |
| Destination reached (ICMP mode) | Type 0 (Echo Reply) | The destination host |
| Router blocks ICMP or no response | Nothing — `* * *` | — |

## Self-check

**In one sentence, why does traceroute work?**

> Traceroute works because every router that drops a packet due to TTL expiry is *required* by the ICMP protocol to send a Type 11 message back to the source — and that reply reveals the router's own IP address, allowing the sender to map the path one hop at a time by progressively increasing the TTL.

## Thesis connection

**How does ICMP relate to your IDS work?**

NSL-KDD has a `protocol_type` feature with values `tcp`, `udp`, and `icmp`. The `icmp` category in NSL-KDD captures several attack patterns:

- **Probe attacks (e.g. `ipsweep`, `nmap`)** — scanners send ICMP Echo Requests to large IP ranges to find live hosts. In NSL-KDD, these appear as many short ICMP connections to many distinct destination IPs with zero or small `src_bytes`.
- **DoS via ICMP (e.g. `pod` — Ping of Death, `smurf`)** — flood-style attacks using ICMP show up as very high `count` (connections to same host in past 2 seconds) with `protocol_type = icmp`.
- **`flag` field in ICMP connections** — NSL-KDD uses `SF` (normal established flow) vs `S0`/`REJ`/`RSTO` etc. For ICMP, `SF` means a complete echo request + reply. An echo request with no reply (`S0`) suggests a scan hitting a dead host.

**Key NSL-KDD features most active during ICMP-based attacks:**
- `protocol_type` = `icmp`
- `count` — spikes sharply during sweeps
- `dst_host_count` — number of distinct destination hosts (high = sweep)
- `dst_host_same_srv_rate` — ICMP sweeps hit many hosts with same "service"
- `src_bytes` — usually very small for ICMP probes (just the echo payload)
