📖 Study — what to cover

W9Transfer learning + CUDA kernels Aug 22 – Aug 28

📖 Study — what to cover

  • Transfer learning — ImageNet pretrained weights encode general visual feature detectors (edges, textures, shapes) in early layers. Fine-tuning replaces the final classification head and retrains it on your domain. Works even when your domain (malware bytes) looks nothing like ImageNet photos — the low-level feature detectors still apply.
  • ResNet18 architecture — residual connections (skip connections) allow gradients to flow unimpeded through identity shortcuts, solving the vanishing gradient problem in 18-layer deep networks. ~11M parameters, fast to fine-tune.
  • CUDA kernel syntax — __global__ void kernel_name(float* data, int n), thread index via threadIdx.x + blockIdx.x * blockDim.x, bounds check if (idx < n), launching with kernel<<>>(args)
  • Memory coalescing — when threads in a warp access consecutive memory addresses, the hardware combines them into a single transaction (coalesced = fast). Strided or random access = multiple transactions = slow. For image processing: access row-major, don't stride.
  • FP16 mixed precision — torch.cuda.amp.autocast() runs forward pass in FP16 (half the memory, 2× throughput on Tensor Cores), keeps master weights in FP32 for numerical stability. Near-free speedup on A100/V100.
